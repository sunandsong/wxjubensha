# 微信群剧本杀小程序

一个在微信群里玩的轻量剧本杀小程序。**房主当主持人**，一幕一幕推进剧情；玩家读本、看线索、投票指认凶手；角色扮演和推理讨论都在微信群里用文字进行——小程序只是把剧本、线索、投票、真相管起来的**辅助工具**。

用**微信云开发**做后端（云函数 + 云数据库实时同步），**不需要自己的服务器**，适合国内使用。

内置三个剧本，房主创建房间时可选：

| 剧本 | 类型 | 人数 | 时长 |
| --- | --- | --- | --- |
| 同学会之夜 | 都市 · 情感 | 3-6 | 约 30-40 分钟 |
| 薄荷小镇 | 小镇 · 悬疑 | 3-6 | 约 35-45 分钟 |
| 深夜咖啡馆（测试） | 测试 · 极简 | 3 | 约 10 分钟 |

> 人数指**玩家**人数，房主作为主持人不占角色、单独算。

## 一、玩法流程

1. **进入即填头像昵称**（首次必填，之后自动记住）。
2. 房主点「**我是房主**」→ 选剧本封面 → 创建房间，得到 4 位房间号，发到微信群。
3. 群友点「**进入房间**」→ 输入房间号加入。
4. 房主点「**开始游戏**」→ 系统给**每个玩家**发一个角色（房主不参与）。
5. 游戏页用 **tab** 组织内容：`第X幕` / `我的角色` / `世界观` / `人物关系`。
6. 主持人**逐幕推进**：点「进入下一幕」。每推进一幕，玩家在「第X幕」tab 看到：
   - 本幕**公开剧情**（旁白，全场可见）
   - **🔒 自己这一幕的私密剧情** + **🎯 本幕盘问任务**（仅本人可见）
   - **搜证区**：本幕的搜查点，每人**限次搜证**，搜到的线索只有自己看到，靠群里交换
8. 末幕结束 → 进入**投票**，玩家在程序里指认凶手（可改票）。
9. 主持人点「**公布真相**」→ 结算页揭晓票数与真相。可「再来一局」重新随机发牌，或「结束游戏」解散房间。

> 扮演、对峙、推理讨论都回**微信群里用文字进行**；小程序负责发剧本、放线索、收投票、揭真相、推进剧情。

## 二、部署步骤（首次约 15 分钟）

### 准备：注册小程序账号
1. 打开 https://mp.weixin.qq.com → 「立即注册」→ 选「小程序」，按引导注册。
2. 在「开发管理 → 开发设置」找到 **AppID**，复制备用。

### 第 1 步：安装微信开发者工具
- 下载：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 安装后用微信扫码登录。

### 第 2 步：导入本项目
1. 克隆或下载本仓库到本地。
2. 微信开发者工具 →「导入项目」→ 目录选本文件夹 → 填你的 AppID → 后端选「小程序·云开发」。
3. 打开 `project.config.json`，把 `appid` 改成你自己的 AppID。

### 第 3 步：开通云开发
1. 工具顶部点「**云开发**」→ 按提示开通（免费基础套餐即可），得到一个**环境 ID**。
2. 打开 `app.js`，把云初始化的 `env` 改成你自己的环境 ID：
   ```js
   wx.cloud.init({
     env: '你的环境ID',   // 例如 'cloud1-xxxxxx'
     traceUser: true,
   });
   ```

### 第 4 步：创建云数据库集合
1. 「云开发控制台 → 数据库」→ 新建集合，名字必须叫 **`rooms`**。
2. 权限设置选「**所有用户可读，仅创建者可写**」；测试期也可用自定义规则 `{ "read": true, "write": true }`（上线再收紧）。

### 第 5 步：上传云函数
- 左侧文件树右键 `cloudfunctions/game` →「**上传并部署：云端安装依赖**」，等待提示成功。

### 第 6 步：预览 / 体验版
- 点「预览」用手机扫码即可玩。
- 发给群友：点「上传」→ 到 https://mp.weixin.qq.com 「版本管理」设为**体验版**，并把群友加进「体验成员」；正式公开需「提交审核 → 发布」。

## 三、目录结构

```
juben-mini/
├─ app.js / app.json / app.wxss      入口与全局（含云环境 env、登录/会话、callGame）
├─ project.config.json               项目配置（填你的 AppID）
├─ utils/
│  └─ scripts.js                     ★ 剧本数据：人物 / 分幕 / 线索 / 真相
├─ pages/
│  ├─ test/                          测试身份页（A/B/C 单机模拟多玩家）—— 上线前移除
│  ├─ index/                         首页：头像昵称 + 我是房主 / 进入房间
│  ├─ room/                          房间：玩家列表、主持人开始/结束游戏
│  ├─ game/                          对局：逐幕剧情 / 私密剧情 / 线索 / 投票
│  └─ result/                        结算：票数与真相揭晓
└─ cloudfunctions/
   └─ game/                          唯一云函数，处理所有对局操作
```

云函数 `game` 支持的动作：`whoami` / `create` / `join` / `leave` / `start` / `advance`（逐幕推进/进投票/公布真相）/ `vote` / `reset` / `dissolve`。

## 四、剧本数据结构（改剧本 / 写新本子看这里）

所有剧情在 `utils/scripts.js`，导出剧本数组。**写新本子照下面这套字段一字不落地配齐**，游戏页会自动按 tab 组织、按可见性发牌。

```js
{
  id, title, subtitle, tag,
  cover: { emoji, gradient },       // 首页封面卡（图标 + 渐变）
  minPlayers, maxPlayers, duration,
  searchPerAct: 2,                  // ★ 每人每幕的搜证次数上限（小本子可设 1）
  intro,                            // 开场背景（房间页/开局公开）
  worldview,                        // ★ 世界观设定 → 公开，单独成「世界观」tab
  victim: { name, desc },
  relations: [                      // ★ 人物关系网 → 公开，单独成「人物关系」tab
    '角色A × 角色B：一句话关系',     //   写角色名即可，展示时自动换成玩家昵称
  ],
  characters: [
    {
      id, name, gender, title,
      isMurderer,                   // 凶手标 true（其余 false）
      // —— 以下进「我的角色」tab，全程不变，仅本人可见 ——
      persona,                      // ★ 人物性格小传：告诉玩家“该怎么演”
      timeline: ['22:50 …', '23:08 …'],  // ★ 个人时间线，精确到分钟，须与线索/真相自洽
      objective,                    // 角色总目标
      secret,                       // ★ 隐藏小秘密：与命案无关但想藏的把柄（非凶手也要有）
      // —— 以下按幕走，进「第X幕」tab，仅本人可见 ——
      actStories: ['第1幕私密', '第2幕私密', '第3幕私密'],  // 每幕私密剧情
      actAsks: [                    // ★ 分幕盘问任务，每幕一组，跟着当幕新线索递进
        ['第1幕该问的'], ['第2幕该问的'], ['第3幕该问的'],
      ],
      actTasks: [                   // ★ 分幕任务：本幕该做的事，全员完成主持人才能推进
        ['做自我介绍', '搜证 1 处', '说出你的行踪'], ['第2幕任务'], ['第3幕任务'],
      ],
    },
    // ...
  ],
  acts: [
    {
      title, narration,             // 本幕公开旁白
      spots: [                      // ★ 本幕搜查地点（按地点搜，可能搜到、可能空手）
        { id: 's1', place: '吧台', clueId: 'c1' },  // 填 clueId = 这里藏着该线索
        { id: 's2', place: '门口' },                 // 不填 clueId = 空地点，搜了一无所获
      ],
      clueIds: ['c1'],              // 仅作真相/参照；搜证以 spots 为准
      hostPrompts: ['冷场时主持人直接抛的问题'],  // ★ 仅主持人可见
    },
    // ...（幕数 = actStories / actAsks 的长度）
  ],
  clues: [
    { id, name, icon, text },       // 线索内容（搜到对应地点才可见）
  ],
  truth: { murderer, title, text },   // 结算页公布；murderer = 凶手角色 id
}
```

### 字段去哪了（游戏页 tab / 可见性）

| 内容 | 位置 | 谁可见 |
| --- | --- | --- |
| `worldview` | 「世界观」tab | 全部 |
| `relations` | 「人物关系」tab | 全部 |
| `persona` / `timeline` / `objective` / `secret` | 「我的角色」tab | 仅本人 |
| `actStories[i]` / `actAsks[i]` / `actTasks[i]` | 「第X幕」tab | 仅本人 |
| `acts[i].narration` | 「第X幕」tab | 全部 |
| `acts[i].hostPrompts` | 「第X幕」tab | 仅主持人 |
| `acts[i].spots`（搜查地点） | 「第X幕」tab 搜证区 | 搜到的人 + 主持人 |

### 搜证规则（按地点 · 限次 · 私密）

- 玩家在「第X幕」tab 的搜证区**按地点搜**，每幕最多搜 `searchPerAct` 处。
- 搜之前只显示 `spots[i].place`（地名）。搜了之后：地点填了 `clueId` → 拿到该线索；没填 → **一无所获**。所以**地点要多于线索**，制造"搜得到也搜不到"的取舍。
- 搜到的线索**只有本人可见**，其余人只看到"谁搜过这个地点"，靠群里交换。
- ⚠️ `spots[i].id` 全局唯一，且要和云函数 `SCRIPT_META.spotsByAct` 里列的地点 id 一一对应。

### 任务与推进门槛

- 每个角色每幕有 `actTasks[i]`（本幕该做的事）。玩家在「第X幕」tab 看到任务清单，做完点 **「完成本幕任务」**。
- **全员完成后主持人才能正常推进**；底部推进按钮会显示 `已完成 X/N`。未全部完成时按钮变灰，主持人点它会二次确认「强行推进」（防 afk 卡死）。
- 完成状态记在房间 `ready` 里，推进到下一幕后自动重置（须重新完成新一幕的任务）。这套是**通用机制，不需配 `SCRIPT_META`**，新本子只要写 `actTasks` 即可。

### 名字替换（重要）

正文里**统一写角色名**（如「阿强」），游戏页会自动把它换成扮演该角色的**玩家昵称**；没人扮演的「公开嫌疑人」(NPC) 保留原角色名。死者名（`victim.name`）不参与替换。所以别在文本里写死玩家昵称。

### ⚠️ 改完必做

1. 角色的 `actStories`、`actAsks`、`actTasks` 长度都要和 `acts` 数量一致（默认 3 幕）。
2. 每幕 `spots` 里有 `clueId` 的地点，其 `clueId` 要能在 `clues` 里找到；建议每幕**地点数 > 线索数**（留空地点）。
3. **同步云函数** `cloudfunctions/game/index.js` 顶部的 `SCRIPT_META`，每个剧本要对齐这几项：
   - `charIds` / `murderer` / `actCount`
   - `spotsByAct`：各幕 `spots` 的 `id` 列表，一一对应（搜证校验用）
   - `searchPerAct`：与剧本里的同名字段一致
   然后 **重新上传部署云函数**，否则发牌 / 推进 / 搜证会出错。

凶手固定，但每局发牌随机、换人扮演仍有新鲜感。

### 范本与完成度

**以「深夜咖啡馆」(`id: test`) 为完整范本**——它配齐了上面所有字段（世界观、人物关系、人物性格、时间线、隐藏小秘密、分幕盘问、按地点限次搜证含空地点）。**写新本子 / 补旧本子，直接照它的结构抄。**

| 剧本 | 完整字段 | 说明 |
| --- | --- | --- |
| 深夜咖啡馆 `test` | ✅ 全 | 范本，新功能都先在这里落地验证 |
| 同学会之夜 `tongxuehui` | ⬜ 待补 | 仅基础字段；缺 worldview/relations/persona/timeline/secret/分幕actAsks/spots |
| 薄荷小镇 `bohezhen` | ⬜ 待补 | 同上 |

> 没补的本子能正常玩：缺的字段在游戏页自动隐藏，搜证退回用 `clueIds` 当地点（无空地点）。补全时照范本逐项加，并同步云函数 `SCRIPT_META`。

## 五、测试身份（单机模拟多玩家）

入口页 `pages/test/test` 提供 **玩家A / B / C** 三个模拟身份：选一个后，客户端会给云函数传一个虚拟 `uid` 覆盖真实身份，于是**一台手机就能切换扮演三个号**测试整局（每页右上角「切换身份」角标）。

推荐：A 当房主创建 → 切 B/C 输房号加入 → 切回 A 开始并逐幕推进。

> 这是临时测试设施。**上线前请移除**：把 `app.json` 的入口改回 `pages/index/index`、删掉 `pages/test`、各页的 `test-switch` 角标，并去掉云函数里 `event.uid` 覆盖身份的逻辑（防止他人伪造身份）。

## 六、常见问题

- **进房间一直转圈/报错**：多半是第 4 步集合 `rooms` 没建，或第 5 步云函数没上传成功。
- **加入后看不到别人**：确认是同一个房间号，且 `rooms` 集合权限可读写。
- **房主进游戏页空白、提示"等待主持人"**：多为改造前开始的旧房间状态不兼容，**重开一局**即可。
- **「云函数不存在」**：回第 5 步重新右键上传部署 `game`。
- **免费额度够吗**：朋友间玩，云开发基础版免费额度绰绰有余。
