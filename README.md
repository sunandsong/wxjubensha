# 同学会之夜 · 微信群剧本杀小程序

一个可以在微信群里玩的轻量剧本杀小程序，3-6 人，单局约 30-45 分钟。
用**微信云开发**做后端（云函数 + 云数据库实时同步），**不需要自己的服务器、也不依赖 Cloudflare**，正好适合国内使用。

内置两个剧本，房主创建房间时可**选择剧本**：
- **同学会之夜**：都市情感，江边别墅坠楼案。
- **薄荷小镇**：小镇悬疑，湖畔别墅 + 松林疗养院谜案。

---

## 一、玩法流程

1. 房主在小程序里**创建房间**，得到一个 4 位房间号，发到微信群。
2. 群友点开小程序，**输入房间号加入**（3-6 人）。
3. 房主点**开始游戏**，系统自动给每人发一个角色和私人剧本。
4. **读本** → **搜证**（轮流翻开线索卡，全员可见）→ 回**微信群里用文字扮演、讨论** → **投票**指认凶手。
5. 房主**公布真相**，结算输赢。可以「再来一局」重新随机分配角色。

> 小程序只是辅助工具，负责发剧本、管线索、收投票和揭晓真相、推动剧情；角色扮演和推理讨论都在微信群里用文字进行。

## 二、部署步骤（第一次需要约 15 分钟）

### 准备：注册小程序账号
1. 打开 https://mp.weixin.qq.com → 右下角「立即注册」→ 选「小程序」，按引导注册（个人也可注册）。
2. 注册后在「开发管理 → 开发设置」里找到你的 **AppID**，复制备用。

### 第 1 步：下载并安装微信开发者工具
- 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 安装后用你的微信扫码登录。

### 第 2 步：导入本项目
1. 把本文件夹 `juben-mini` 放到电脑上任意位置。
2. 打开微信开发者工具 →「导入项目」。
3. 目录选择 `juben-mini` 文件夹；AppID 填你刚才复制的那个；后端服务选「小程序·云开发」。
4. 打开后，编辑 `project.config.json`，把 `"appid": "请替换为你的小程序AppID"` 改成你的真实 AppID（或在导入时填好即可）。

### 第 3 步：开通云开发
1. 在开发者工具顶部点击「**云开发**」按钮 → 按提示**开通**（选免费的基础套餐即可）。
2. 开通后会得到一个**环境 ID**。
3. 打开 `app.js`，把云初始化改成你的环境 ID（如果只有一个环境，可不填，使用默认环境）：
   ```js
   wx.cloud.init({
     env: '你的环境ID',   // 例如 'cloud1-xxxxxx'
     traceUser: true,
   });
   ```

### 第 4 步：创建云数据库集合
1. 进入「云开发控制台 → 数据库」。
2. 新建一个集合，名字必须叫 **`rooms`**。
3. 点该集合的「权限设置」，选「**所有用户可读，仅创建者可写**」或「自定义安全规则」。
   推荐自定义安全规则填：
   ```json
   { "read": true, "write": true }
   ```
   （仅供测试/朋友间玩，方便起见。上线正式运营再收紧。）

### 第 5 步：上传云函数
1. 在开发者工具左侧文件树里，右键 `cloudfunctions/game` 文件夹。
2. 选择「**上传并部署：云端安装依赖（不上传 node_modules）**」。
3. 等待几十秒，提示上传成功即可。

### 第 6 步：预览 / 真机调试
- 点工具栏「预览」，用手机微信扫码，即可在手机上玩。
- 想发给群友体验，点「上传」提交版本，再到 https://mp.weixin.qq.com 「版本管理」里设为体验版或提交审核发布。

## 三、目录结构

```
juben-mini/
├─ app.js / app.json / app.wxss      小程序入口与全局配置
├─ project.config.json               项目配置（记得填 AppID）
├─ sitemap.json
├─ utils/
│  └─ scripts.js                     剧本数据（人物、线索、真相）★改剧本看这里
├─ pages/
│  ├─ index/                         首页：创建/加入房间
│  ├─ room/                          大厅：玩家列表、房主开始
│  ├─ game/                          对局：读本 / 搜证 / 投票
│  └─ result/                        结算：揭晓真相、再来一局
└─ cloudfunctions/
   └─ game/                          唯一的云函数，处理所有对局操作
```

## 四、想改剧本？

所有剧情都在 `utils/scripts.js` 里。该文件导出一个剧本数组，每个剧本结构清晰：
- `id` / `title` / `subtitle` / `tag`：剧本标识与首页展示信息
- `intro`：开场背景
- `characters`：每个角色的公开身份、私人故事、秘密、目标，`isMurderer: true` 标记凶手
- `clues`：搜证卡（线索）
- `truth`：最终真相文本，`murderer` 指定真凶 id

**新增一个剧本**：在 `utils/scripts.js` 里照着已有结构再写一个对象，加进底部的 `SCRIPTS` 数组即可，首页会自动出现这个选项。

⚠️ 如果新增剧本，或改了某个剧本的角色 id / 凶手，必须**同步**修改云函数
`cloudfunctions/game/index.js` 顶部的 `SCRIPT_META`（每个剧本的 `charIds` 和 `murderer`），
然后**重新上传云函数**，否则发牌会出错。

每个剧本的凶手目前是固定的。换人扮演同一角色也有新鲜感；想做「凶手随机」的进阶版可以再找我加。

## 五、常见问题

- **进房间一直转圈/报错**：多半是第 4 步集合没建，或第 5 步云函数没上传成功。
- **加入后看不到别人**：确认大家用的是同一个房间号，且 `rooms` 集合权限是可读写。
- **「云函数不存在」**：回到第 5 步重新右键上传部署 `game` 云函数。
- **免费额度够吗**：朋友间偶尔玩完全够，云开发基础版免费额度对小流量绰绰有余。

祝玩得开心！🕵️
