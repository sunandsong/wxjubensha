两种元素，四个动作

  • 光——照亮哪里，注意力就在哪里：候场只有就位的人亮；剧情页全屏只照亮你的故事那一块（外面压暗 55%）；指认时光只留在被指认的人身上；公布真相 = 破晓，整屏从深夜渐变成黎明。
  • 线——推理的具象：候场的线穿过所有席位，就位 = 在你头像上绕一圈打结；关键句下的划线是线自己长出来的；投票 = 其余人的线一起牵向你选的人；真相 = 万线归一。
  • 光的冷暖随本切换（顶部开关试试）：悬疑本冷紫月光，欢乐本暖金阳光——同构不同温。tag 含「恐怖」可以再压暗 + 线变细颤动。
  • 零 emoji 装饰、零场景道具、零拟物——这版的克制本身就是风格。

与真实流程逐页对应

  • room：准备 = 线打结（ready 字段驱动 SVG 描边动画）；「全员就位」文案改为「线连齐了，故事才开始」。
  • game：开局播一次「光扫过 + 线写出剧本名」过场（现有 transShow 钩子换皮）；玩家屏 = 被照亮的故事卡（box-shadow 一行实现外部压暗）；主持人发群暗卡照旧。
  • voting：点谁 = 其余人的线牵向谁 + 光聚到谁（roster 位置已知，4~7 条 line 的 SVG 即可）；可改票 = 线重新牵。
  • result：主持人「公布真相」→ 破晓渐亮 + 结果行升起；玩家等待页 = 一行字「天快亮了，等主持人揭晓…」。

落地

  • 核心技术就三样：SVG 描边动画（线）、box-shadow 外部压暗（光斑）、背景渐变过渡(破晓)——小程序全支持，性能压力小于剧场版。
  • 照旧只动 wxml/wxss + 20 行主题函数（冷暖光/线色 ← cover.gradient、tag）。
  • 顺序：②照亮的故事卡 + 开场光影 → ③万线归一 + 破晓 → ①线穿席位。

口径提醒：对外文案走「推理/真相大白」，抽象风没有任何敏感意象，备案口径最安全的一版。
