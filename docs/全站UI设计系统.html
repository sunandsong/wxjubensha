DESIGN SYSTEM · V1

夜幕紫 设计系统 · 全功能一套 UI

一句话定位：一座深夜的游戏厅——底盘永远是夜幕紫，每个游戏占一种光色。不做强主题（剧场/古堡都会在某类本子上出戏），沉浸感交给三样东西：剧本自己的封面与文案、每个游戏的主题光、以及少量「仪式时刻」动效。四条铁律贯穿所有页面：读字区素颜 · 花活只在过场 · 一屏一个主按钮 · 状态必有反馈。

01色彩 Tokens

底与玻璃永远不变；游戏光谱是唯一随场景变化的东西——进入哪个游戏，哪个色就是那一屏的 --ac（按钮、座灯、选中态、高亮划线全跟它走）。语义色独立于光谱，不挪用。

05主题色应用规则

元素用什么色说明
页面底 / 导航栏 / 玻璃卡bg1 / bg0 / glass（固定）任何游戏都不许改底色——统一感的根
主按钮 / 选中态 / 高亮划线 / 进度当前游戏 --ac一屏只有一个渐变主按钮
席位就位灯warn 金（全游戏统一）「准备好」是跨游戏的同一动作，色不随游戏变
真凶 / 出局 / 危险bad 红（固定）语义色不挪用游戏色
剧本详情 / 对局的氛围点缀剧本 cover.gradient剧本自己的颜色只出现在剧本杀内页

06仪式时刻（唯一允许的花活）

  • 剧本杀 · 开演：全屏过场——剧本封面渐显 + 剧名 +「故事开始」，1.2s 收场（现有 transShow 换皮）。
  • 卧底 · 揭晓：卧底头像 0.8s 变色翻转 + 词对亮出；狼人 · 天亮：夜幕带一次月落日升的明暗过渡。
  • 炸弹 · 爆炸：已有的全屏地震 + 💥 保留——它就是规范本身。
  • 结算通用：结果行逐行升起（0.45s 阶梯延迟），抓对撒花、抓错安静——喜怒有别。
  • 以上之外不加动效。读字区（剧情卡/词卡/汤面）永远静止。

07落地清单（与现状的差距）

  • 零改动就达标：hub、数字炸弹（本来就是规范来源）。
  • 只调 wxss：profile（已达标 95%）、剧本列表/详情（卡片圆角与 chip 统一）、room（席位组件化 + 开始按钮 dim 态——刚已修）。
  • wxss + 少量 wxml：game/result（剧情卡素颜化、结果行升起动效）、spy（词卡 + 粉色 --ac）、wolf（夜晚带 + 角色卡 + 橙色）、soup（汤面卡 + 蓝色）。
  • 建议顺序：① room/game/result（主链路）→ ② spy/wolf/soup 配色统一 → ③ 仪式时刻逐个补。
  • 所有页面共用一份 styles/tokens.wxss（色值 + 按钮 + 卡 + 席位），新页面 import 即得。

单主题说明：App 是「深夜游戏厅」，暗色是身份不是缺省——不做浅色模式；对外文案口径照旧（推理/派对，不带敏感词）。
