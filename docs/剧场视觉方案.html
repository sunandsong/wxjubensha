换本 = 换戏，UI 一行不改

随剧本自动变数据来源示例（照相馆 → 浪浪山）
主题色（按钮/座灯/追光/划线/幕布）cover.gradient（已有字段）紫蓝冷色 → 橙金暖色
开演过场（emoji/剧名/氛围词）cover.emoji + title + tag「谜底藏在其中」→「全体入戏，不许笑场！」
彩蛋强度tag 含「欢乐」/「悬疑」悬疑：压暗克制 → 欢乐：开演即撒花
恐怖特效包（可选）tag 含「恐怖」叠加幽影古堡的红眼/雷闪/烛光呼吸

剧场语言 → 真实流程逐页对应

  • room 候场：剧本卡→「今夜上演」牌匾（小灯泡随主题色呼吸）；准备 = 入座亮座灯；开始 = 「开演」。玩家在剧场语境里是演员——和剧本杀"扮演"的本质严丝合缝。
  • game 剧情：开局播一次大幕拉开（幕布染主题色，欢乐本落彩带）；玩家屏 = 素颜剧情卡（划线色随主题）；主持人多「发群内容」暗卡，一键复制照旧。
  • voting 投票：roster → 舞台追光：点谁追光打谁、其余暗下去（悬疑冷白光/欢乐暖黄光）；可改票、已投进度照旧。
  • result 结算：谢幕——「本剧谢幕」+ 演员表逐行升起（角色 · 玩家 · 得票，真凶标红），欢乐本撒花；玩家端等待页 = 「幕间休息，等导演揭晓真相…」。

落地

  • 照旧只动 wxml/wxss + 一个主题解析函数（读 cover.gradient/tag → 输出 CSS 变量，约 20 行 js），对局逻辑零改动。
  • 顺序：②大幕+剧情卡 → ③追光+谢幕 → ①牌匾候场。恐怖特效包最后做，按 tag 挂载。
  • 以后上新本：填好封面渐变和 tag 就自动有自己的"戏"，零适配成本——这是这版相对强主题方案的决定性优势。
