案卷五要素 → 页面映射

  • 委托书（候场 room）：白卡片换米黄纸纹 + 虚线毛边；房间号 → 右上角红框「案件编号」；准备 = 席位盖红手印；开始按钮 → 「开案」。
  • 卷宗过场（game 分幕切换）：现有淡入淡出升级为「卷宗·第 N 章」+ 「启封」红章砸落（纯 CSS 动画，逻辑不变）。
  • 手记信纸（我的剧情）：pre 文本装进信纸卡，横线纸纹 + 高亮句改红笔划线（现有 hl 分段数据直接复用）。
  • 证物袋（搜证）：搜查点 = 牛皮纸证物袋 + FORENSIC 封条；点「搜查」= 撕封条开袋；「已搜:XX」→ 🫆 指纹小标。线索本 = 软木板 + 拍立得。
  • 指认与结案（投票 / result）：投票列表 → 软木板通缉照 + 📍红图钉 + 红线牵到「真凶?」；主持人公布 = 「结案」大红章砸落，抓对保留撒花。玩家等待页也换「卷宗封存中…」文案。

落地

  • 只动 wxss + wxml 包装层：room / game / result 的数据流、云函数、主持人复制发群逻辑一行不改。
  • 顺序建议：②手记信纸+证物袋（游戏页最常看）→ ③指认板+结案章（高光时刻）→ ①委托书候场。
  • 撕封条 / 盖章 / 手印均为 CSS transition + keyframes，与数字炸弹页的爆炸动效同量级，无性能压力。
  • 指认红线：小程序端用 4 条预置角度的 view 细条即可（mockup 里是 SVG），不必引入 canvas。

口径提醒：对外文案沿用「推理」系说法（推理开局/侦探团/案件），与个人主体备案口径一致，不出现敏感词。
