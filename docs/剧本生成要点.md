# 剧本生成要点（当前版 · v3）

> 单一事实来源。给我一段剧情/一个现成剧本，我按本文产出可玩新本。
> 样板：`test`(巷往咖啡馆)、`shiguang`(拾光·打烊之后)；欢乐本变体：`nanji`(南极企鹅)、`shiqian`(圣火)。
> 内容铁律详见同目录 [新本生成模板.md](新本生成模板.md)，**本文是其「§五接入」的最新替代 + 内容要点提炼**。

---

## 一、玩法模式

- **角色=真人**：房主当主持人（不演角色、只推流程、群里公布），其余 N 人各演 1 角色，**必须满 N 人**才能开始（N=角色数）。
- **单幕 autoClues**：开场动画(剧本名) → 每人读角色卡 → 线索随幕自动公开（不搜证） → 主持人「一键复制」把开场白+群活动+线索发微信群 → 群里讨论互指 → 回小程序投票 → **投满才公布真相**。
- **吃瓜群众**：游戏开始后进来的人只围观第一幕、无任何操作按钮。
- **结算署名动态**（用 `victim.name` 等，不要写死）。

---

## 二、内容铁律（迭代踩坑总结，最重要）

1. **线索全是物证、一人一条**：N 个嫌疑人就 N 条线索，每条是一件**现场物证**，各把一个人坐实"到过现场/有嫌疑"。
2. **动机不进线索**：动机三人都有，放在 `intro / worldview / relations / 角色卡`，**绝不写进线索**。
3. **物证不点名、不给答案**：线索只描述客观物证，不能出现"谁手上有伤""跟 X 摊牌的录音"这类广播式 giveaway。
4. **三人物证等重**：每条都让对应的人看着都像；谁明显弱就加强。**凶手的线索 = 旁证/物证，不是铁证**；红鲱鱼给"看似指向"的线索，但**开脱写进他自己的 `actStories`**（不在公开线索里）。
5. **凶手在真相里才锁定**：靠"凶器来历 + 他的破绽"对上一个人；致命细节只放凶手角色卡 + 真相收尾，**不进公开线索**。
6. **时间线闭环**：被害人"一个一个单独算账"（造无目击）；明确**谁先发现尸体**；每人对外一句行踪——**凶手撒谎、其余属实**。
7. **白描叙述**：角色故事用客观白描（看得见的动作/物件），**不是内心独白**。
8. **篇幅均衡**：各角色卡长度相当，性格各自鲜明。
9. **红鲱鱼要有真动机**：非凶手也要有像样的杀人动机 + 一个能自圆其说的不在场/解释。
10. **性别中立**：角色尽量不锁死性别（`gender:""`），称呼中性。
11. **去套路**：不要"破绽/把柄/抵死不认/我得演下去"这类**狼人杀腔 + 元叙述**；角色不"知道自己是凶手在演戏"，只客观陈述。
12. **欢乐本变体**（南极/圣火）：把"凶杀"软化成"偷东西/闹乌龙"，"凶手"= 偷石头/偷烤肉的那个；基调轻松、动物/史前设定，但**线索仍是一人一条物证、仍靠推理**。

## 三、角色卡（`actStories[0]`）& 真相

- **角色卡 5 段**（段间 `\n\n`，重点句 `**…**` 成对高亮）：①我是谁 ②和被害人的恩怨 ③今晚行踪+我的物证/嫌疑 ④甩锅另两人（各一句） ⑤结尾伏笔（括号）。凶手额外给一句**对外撒谎口径**。
- **真相小说体**：被害人其人 → 三人各自把柄+动机 → 那一夜逐个算账 → 自然揭凶 → **反转/主题句收尾**。不要"线索分析①②③"。反转二选一：被害人遗书 `letter`，或让真相自己承担、`letter` 留空。

## 四、数据字段（`utils/scripts.js`）

```js
const SCRIPT_XXX = {
  id, title, subtitle, tag,
  cover: { emoji, gradient, image: "" },   // image 填 cloud:// fileID（见§六）；空则显示 emoji+渐变
  minPlayers, maxPlayers,                   // = 角色数 N
  duration, autoClues: true,
  intro, worldview,
  victim: { name, desc },                   // 结算署名用
  relations: [ "A → B：恩怨/动机", ... ],   // 公开关系网=动机池
  characters: [ {
    id, name, gender:"", title, isMurderer,
    persona:"", timeline:[], secret:"",
    actAsks:[["盘问别人一句"]], actTasks:[["本幕要做的事"]],
    objective:"", actStories:[ "§三 5 段结构" ],
  }, ... ],                                  // N 个
  acts: [ { title, narration, clueIds:["x1","x2","x3"],
            hostPrompts:["冷场直接问"],
            host:{ cast:[一句话介绍每人+嫌疑], story:开场白, activities:[群活动4步] } } ],
  clues: [ { id, name, icon, text(纯物证) } ],  // 一人一条
  truth: { murderer(角色id), title, text(小说体), letter("" 或 遗书) },
};
```

> ⚠️ 别让凶手永远排在 `characters[]` 第一个——展示端虽已按房间种子打乱（[game.js](../pages/game/game.js) roster shuffle、[index.js](../pages/index/index.js) 详情 roster），但写数据时也别固定顺序。

## 五、接入步骤（云数据库架构，**已不用手改 SCRIPT_META**）

剧本现在运行时从**云数据库**取（[utils/scriptStore.js](../utils/scriptStore.js) → 云函数 `getScripts`），打包的 `scripts.js` 只做兜底；发牌元数据由云函数 `getMeta` **自动从剧本数据派生**（云库→种子→旧硬编码三重兜底）。所以加新本：

1. **写数据**：`utils/scripts.js` 加 `SCRIPT_XXX` → 加进 `SCRIPTS` 数组 → 加进 `module.exports.list`（首页展示）。
2. **重新生成种子**（灌库+兜底数据，在项目根目录跑）：
   ```bash
   node -e "const fs=require('fs');const S=require('./utils/scripts.js');fs.writeFileSync('cloudfunctions/game/scriptsSeed.json',JSON.stringify(S.list.map(s=>Object.assign({_id:s.id,shown:true},s)),null,2))"
   ```
   （即：把 `scripts.js` 的 `list` 每条加 `_id/shown` 写进 `cloudfunctions/game/scriptsSeed.json`。）
3. **部署云函数**：右键 `cloudfunctions/game` →「上传并部署」（带上新种子）。
4. **灌库**：云函数云端测试 `{"action":"seedScripts"}` → `seeded:N`（让数据库也更新；不灌也能跑，会回退种子）。
5. **自检**：建房后房间页显示「需 N 名玩家」= N 即成功。
6. （`cloudfunctions/game/index.js` 里的 `SCRIPT_META` 现在只是最终兜底，**新本不必再加**；`getMeta` 会从数据自动算 charIds/murderer/cluesByAct/actCount/searchPerAct。）

## 六、封面（云存储 cloud://）

- 封面**不放代码包**（主包 2MB 限制）。`cover.image` 填云存储 `cloud://...` fileID，`<image>` 原生支持。
- 上传方式：把图放 `/assets/<id>.jpg` → 临时把 `cover.image` 指过去 → 进 **test 页点「⬆️ 上传封面到云存储」**（[pages/test/test.js](../pages/test/test.js)）自动传云存储 + 写 fileID 进库 → Console 打印 `cloud://...` → 把 fileID 硬编码回 `cover.image` → 删本地图 → 重新生成种子。
- 当前 env/bucket 前缀：`cloud://cloud1-d6g6wknyy4d198022.636c-cloud1-d6g6wknyy4d198022-1446823337/covers/<id>.jpg`
- AI 封面提示词：横版 3:2、无文字、底部留暗压标题；悬疑本走写实电影感，欢乐本走可爱卡通。
- 分享卡片缩略图 = 当前本封面（[room.js](../pages/room/room.js) 用 `getTempFileURL` 转 https）；`/assets/share.jpg` 保留作兜底分享图，勿删。

## 七、你给我什么，我就能生成

最少给：①场景&被害人（在哪、谁、什么性格、怎么死、密室怎么成立）②N 个角色（身份、与被害人恩怨、今晚行踪、是不是凶手）③凶手手法（凶器、怎么毁证、什么破绽锁定他——别写进公开线索）④反转/主题。
或者**直接丢一个现成剧本/小说**，我按本文全拆解改写。

## 附：项目关键事实（换电脑速览）

- 微信小程序 + 微信云开发（云数据库 `rooms`/`scripts`、云函数 `game`、云存储）。**后端必须腾讯云开发，不能 Cloudflare**；小程序运行时**不能直连 Claude/海外 API**。
- 小程序名：**群本杀**；类目：工具·信息查询（避开"剧本杀实体"资质）。
- 用户提交习惯：直接提交到 `main`，push 要明说；云函数改动需在开发者工具「上传并部署」才生效。
