# 新剧本制作指南

> 基于「巷往咖啡馆」的实战经验整理。做新本时照着这份走，能少踩很多坑。

---

## 一、剧情设计（最关键）

- **先定真相，再倒推**：先想清楚「谁杀的、怎么杀、为什么」，再给每个角色编故事、撒线索。别先写人再凑案。
- **真凶要最不像**：让动机最重、最可疑的那个当**红鲱鱼**（如阿强：假账 + 毒药货仓 + 删录音），真凶看着最干净（如小美）。破案靠**组合线索**，不是单条直接指向。
- **线索点状、分散**：每条线索是一个「点」，分到不同人身上，别全烧向一个人。让玩家自己拼。
- **每幕一个收获**：每幕推进都丢一个新故事 / 新线索。结构尽量统一（咖啡馆用「自己 → 旁人 → 翻盘 → 那只杯子」）。
- **第一视角只写自己**：玩家手里只有自己的故事，**只描述事件、不写心理活动**；想要的信息靠提问别人或嫁祸别人得到。
- **动机 / 手段 / 机会 三件套**：每个嫌疑人都要在这三方面有戏；真凶的「机会」藏巧一点（咖啡馆：毒在杯底 → 让「案发时不在场」失效）。
- **角色可男可女**：名字用玩家昵称替换、剧情尽量写中性，适配面更广。
- **结尾留余味**：给个反转 / 情感落点（老板查出癌症 + 遗书），一局结束有回味。

## 二、数据结构（照着填）

新本 = 往 `utils/scripts.js` 的 `SCRIPTS.list` 加一个 SCRIPT 对象。字段：

```js
{
  id: 'xxx',                 // 唯一 id，云函数 META 的 key 要与它一致
  title: '剧本名',
  subtitle: '一句话副标',
  tag: '悬疑 · 人性',
  cover: {
    emoji: '☕',
    gradient: 'linear-gradient(135deg,#8a5a30,#3a2415)',
    image: '/assets/xxx.jpg',  // 可选：封面图（没有就用 emoji+渐变）
  },
  minPlayers: 3, maxPlayers: 3, duration: '约 60 分钟',
  autoClues: true,           // 线索随幕自动公开（推荐）

  truth: {
    murderer: 'xiaomei',     // 真凶角色 id
    title: '真相标题',
    text: '通用真相旁白（所有人可见）',
    letter: '遗书 / 压轴文本（仅主持人，可下载成图）',
  },

  characters: [
    {
      id: 'xiaomei', title: '咖啡馆店员', gender: '',  // gender 留空，按玩家性别走
      actStories: [
        '第一幕：第一视角故事…',
        '第二幕：…',
        '第三幕：…',
      ],
    },
    // 其余角色…
  ],

  acts: [
    {
      title: '第一幕 · 标题',
      clueIds: ['t1'],        // 本幕公开哪些线索
      host: {
        story: '主持人本幕剧情（小说式）',
        activities: ['① 自我介绍…', '② 公开线索…', '③ 讨论…'],
      },
      hostPrompts: ['冷场时问 A…', '问 B…'],  // 可选
    },
    // 其余幕…
  ],

  clues: [
    { id: 't1', name: '线索名', icon: '☕', text: '线索内容', img: '/assets/clues/t1.jpg' /* 可选 */ },
    // …
  ],
}
```

角色照片命名约定：`/assets/avatars/{角色id}_{m|f}.jpg`（每个角色备男、女两版，按玩家性别显示 / 下载）。

## ⚠️ 三、最大的坑：客户端与云函数要对齐

`utils/scripts.js` 和 `cloudfunctions/game/index.js` 的 `SCRIPT_META` **必须对齐**，否则发牌、推进、真相会错乱。

云函数 META 里每个本子要写：

```js
SCRIPT_META = {
  xxx: {
    murderer: 'xiaomei',                 // 与 scripts.js 的 truth.murderer 一致
    charIds: ['xiaomei','aqiang','laoli'],// 所有可发牌角色 id
    actCount: 3,                          // 幕数，与 acts.length 一致
    cluesByAct: [['t1'], ['t2','t3','t4','t5'], ['t6','t7','t8','t9','t11']], // 每幕线索
  },
}
```

对齐项：**`murderer` / `charIds` / `actCount` / `cluesByAct`**。

**改完云函数务必重新部署**：开发者工具右键 `cloudfunctions/game` → 「创建并部署」。
> 部署前确认 `project.config.json` 的 `cloudfunctionRoot` 是顶层字段、且已给 `cloudfunctions` 目录绑定云环境（否则会报「请选择一个云环境」）。

新本做好后，记得在 `scripts.js` 的首页列表里**取消隐藏**（目前同学会 / 薄荷小镇是隐藏状态，有注释）。

## 四、做图经验

- **AI 出图不写中文**——中文字会糊成乱码。
  - 要**文字准确**的（遗书 / 诊断书 / 现场平面图）→ 用 **canvas 代码画**（项目里已有现成实现，可复用：`pages/result/result.js` 的 `_drawLetter` / `_drawReport`、`pages/game/game.js` 的 `_drawMap`）。
  - 纯**氛围图**（封面 / 海报 / 线索物件）→ AI 出图即可，别让它写字（负面词加 `text, words, watermark`）。
- **压缩**：图进项目前用 `sips` 压（缩长边 + 转 JPG），别塞大图——小程序主包限 2MB。
  - 例：`sips -Z 800 -s format jpeg -s formatOptions 78 in.png --out out.jpg`
- **角色照按性别两版**（`xxx_m.jpg` / `xxx_f.jpg`）。
- **分享卡片图**比例 **5:4**（如 1000×800），主体居中防裁（微信卡片按 5:4 裁切）。
- **线索图分类**：实物 / 场景（咖啡杯、货仓）配图；抽象 / 推理 / 录音类（被删录音、谁碰得到杯子）纯文字就好。

## 五、主持人手卡

- 每幕给主持人一份**群活动清单**（`acts[].host.activities`）+ **追问钩子**（`hostPrompts`）。
- 写**具体动作**：「让大家轮流自我介绍」「公开线索 t2」「讨论谁碰过杯子」——
  别写「活跃气氛」「引导讨论」这种模糊词。
- 线索、提示在游戏里都可「复制文字 / 下载图片」发群。

## 六、新本制作 Checklist

1. [ ] 定真相：真凶、手段、动机、机会。
2. [ ] 设计每个角色的分幕第一视角故事（只写事件，不写心理）。
3. [ ] 设计线索：点状、分散、不全指向真凶；标好每幕公开哪些。
4. [ ] 写主持人每幕活动清单 + 追问钩子。
5. [ ] 写真相旁白 + 遗书 / 结尾反转文本。
6. [ ] 往 `utils/scripts.js` 加 SCRIPT，首页列表取消隐藏。
7. [ ] 往 `cloudfunctions/game/index.js` 加 `SCRIPT_META`（对齐 4 项）。
8. [ ] **部署云函数**。
9. [ ] 出图：封面 / 海报（AI）、角色照两版（AI）、需要文字的用 canvas；全部压缩。
10. [ ] 用测试身份（`app.js` 把 `TEST_ENABLED` 改 `true`）跑一局全流程，测完改回 `false`。

## 七、可直接复用的技术资产

无需重写，新本套用即可：

- 按玩家性别显示 / 下载角色照片
- 线索「复制文字 / 下载图片」、主持人手卡复制
- canvas 画「文字图」（遗书 / 诊断书 / 现场平面图）
- 分享小程序卡片进房（群友点卡片自动加入；首次先走授权向导再进房）
- 三步授权向导（头像 / 昵称 / 性别）+ 可编辑资料
- 全套特效（分幕过场、结算撒花、卡片入场、按钮反馈、封面 Ken Burns）
- 测试身份多人模拟（总开关 `TEST_ENABLED`，仅非正式版显示入口）

> 这些都做成了通用机制，新本主要工作就是**填剧情数据 + 出图 + 对齐云函数 META + 部署**。
