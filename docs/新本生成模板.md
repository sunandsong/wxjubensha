# 10 分钟单幕本 · 生成模板

> 以「巷往咖啡馆」(id: `test`) 为样板。给我一个剧本，我按本文结构直接产出一个新本。

## 一、玩法模式（先理解再套）

- **角色 = 真人**：房主当主持人（不演角色、只推流程、群里公布），其余 N 人各演 1 个角色。**必须满 N 人才能开始**（N = 角色数）。
- **单幕**：开场动画(剧本名) → 每人读自己的角色卡 → 线索随幕自动公开（`autoClues`，不搜证）→ 主持人把「开场白+群活动+线索」一键复制发微信群 → 群里讨论/互指 → 回小程序投票 → **投满才能公布真相**。
- **结算分流**：玩家端只看「去群里等待主持人公布真相」；主持人端看投票结果+真相+信，一键复制发群公布。
- **反转**：被害人临终亲笔信（`truth.letter`），压轴揭一个「谁都没料到」的结局（样板里张涛癌症晚期、原谅三人、把店留给凶手）。

## 二、要改的两个文件（必须同步！）

1. `utils/scripts.js` —— 剧本内容（角色卡、线索、真相、信…）
2. `cloudfunctions/game/index.js` 里的 `SCRIPT_META[id]` —— 发牌/推进配置（charIds、murderer、actCount、cluesByAct）
3. ⚠️ 改完 **云函数必须重新部署**（右键 `cloudfunctions/game` →「上传并部署：云端安装依赖」），否则发牌/推进不生效。
4. 首页展示：`module.exports.list` 控制（现在只放当前本）。

## 三、scripts.js 结构（逐字段）

```js
const SCRIPT_XXX = {
  id: "xxx",                       // 唯一 id，和 SCRIPT_META 的 key 一致
  title: "本名",
  subtitle: "一句话钩子",
  tag: "悬疑 · 人性",
  cover: { emoji: "☕", gradient: "linear-gradient(135deg,#a,#b)", image: "/assets/share.jpg" },
  minPlayers: 3, maxPlayers: 3,    // = 角色数
  duration: "约 15 分钟",
  autoClues: true,                 // 单幕本固定 true（线索随幕公开、不搜证）

  intro: "首页选本简介，3~4 句，交代场景+命案+钩子",
  worldview: "开场世界观长文，交代环境/被害人来历/把三人逼到墙角/命案现场（密室）",
  victim: { name: "被害人名", desc: "被害人是谁、什么性格、为什么招恨" },

  relations: [                     // 公开关系网，每条「A → B：恩怨」
    "被害人 → 角色1：……（动机钩子）",
    "被害人 → 角色2：……",
    "被害人 → 角色3：……",
    "角色1 × 角色2：……",
  ],

  characters: [ /* 见下，N 个 */ ],
  acts: [ /* 单幕只 1 个，见下 */ ],
  clues: [ /* {id,name,icon,text}，见下 */ ],
  truth: { /* 见下 */ },
};
```

### 角色 character（每个角色一份）

```js
{
  id: "rolea", name: "角色名", gender: "", title: "身份（如 咖啡馆店员）",
  isMurderer: false,             // 凶手 true
  persona: "", timeline: [], secret: "",   // 单幕模式留空
  actAsks:  [["盘问别人的一句话"]],          // 单幕只用 [0]，给主持人/玩家提问灵感
  actTasks: [["本幕要做的事1","事2"]],       // 单幕只用 [0]
  objective: "",
  actStories: [
    // ★核心：玩家看到的角色卡。单幕=1 张完整卡，结构固定：
    // ① 我是谁+背景  ② 和被害人的恩怨(动机)  ③ 今晚行踪+作案/我的嫌疑
    // ④ 甩锅两个方向（点名另两人各自的嫌疑）  ⑤（结尾括号：给反转埋的伏笔）
    "第一段……养家。\n\n" +              // 段落之间用 \n\n（空行间距）
    "第二段(动机)……。\n\n" +
    "第三段。**这里是高亮重点句（凶手=作案手法+不在场；平民=最关键的辩词或最大嫌疑）。**\n\n" +
    "**甩锅方向一：另一人最有机会/料最多……。**\n**甩锅方向二：第三人接触最直接……。**\n\n" +
    "（结尾伏笔：被害人最近反常——跑医院、深夜写东西、见人锁抽屉……为那封信铺垫。）",
  ],
}
```

**重点高亮**：正文里用 `**……**` 包住的句子会暖黄加粗高亮。约定每人高亮 3 句 = 自己最关键的 1 句 + 甩锅的 2 个方向。

### 幕 act（单幕 1 个）

```js
{
  title: "幕标题（如 密室惊变）",
  narration: "本幕念白：现场+命案+谜面，3 句",
  clueIds: ["x1","x2","x3"],     // 本幕公开的线索 id
  hostPrompts: ["冷场时主持人能直接问的一两句"],
  host: {                         // 主持人专属
    cast: [                       // 一句话介绍每人 + 嫌疑点（主持人引导用）
      "角色1 · 身份：最有机会——……。",
      "角色2 · 身份：料最多——……。",
      "角色3 · 身份：接触最直接——……。",
    ],
    story: "主持人开场白（小说式，2~3 句，反锁密室+毒+三人之中）",
    activities: [                 // 群活动步骤（当前 4 步）
      "① 轮流自我介绍 + 解释线索：……",
      "② 群里初投：……",
      "③ 当众反驳：……",
      "④ 最终投票：回小程序投票、公布真相。",
    ],
  },
}
```

### 线索 clue

```js
{ id: "x1", name: "线索名", icon: "☕", text: "线索内容（指向手法/来路/嫌疑）" }
```
约定 3 条：**命案本身 / 手法（毒在哪、怎么下的）/ 来路（凶器或毒的出处，能锁定凶手）**。

### 真相 truth

```js
truth: {
  murderer: "rolea",            // 凶手角色 id
  title: "本名",
  text:                          // 公开真相：凶手是谁+动机+手法+为什么锁定他+其他人的烟雾弹
    "凶手是……。\n\n动机……。\n\n手法+不在场……。\n\n真正锁定的关键证据……。\n\n其他人各有把柄但都不是凶手。",
  letter:                        // ★反转·被害人亲笔信（只发主持人，压轴念）
    "有些话我活着说不出口……（揭一个反转：被害人其实另有隐情/将死/忏悔/原谅）……\n\n——被害人名",
}
```

## 五、你给我什么，我就能生成

最少给我这些，我来补全结构、写卡、设高亮、配两处文件：

1. **场景 & 被害人**：在哪、被害人是谁、什么性格、怎么死的（密室/手法）。
2. **N 个角色**：每人——身份、和被害人的恩怨(动机)、今晚行踪、是不是凶手。
3. **凶手的手法**：毒/凶器是什么、怎么不在场、什么证据能最终锁定他。
4. **反转**：被害人那封信要揭什么（忏悔/隐情/原谅…）。
5. （可选）线索 3 条的方向；没有我按「命案/手法/来路」默认配。

或者——**直接把一个现成剧本丢给我，我负责拆解、改写成这套结构**（含段落间距、重点高亮、主持人发群内容、真相+信、两处文件同步）。
