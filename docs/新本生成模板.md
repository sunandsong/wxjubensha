# 10 分钟单幕本 · 生成套路（v2）

> 以「巷往咖啡馆」(`test`)、「拾光·打烊之后」(`shiguang`) 为样板。
> 给我一个剧情/剧本，我按本文套路直接产出一个可玩的新本（含两文件同步）。

---

## 〇、玩法模式（先理解）

- **角色=真人**：房主当主持人（不演角色、只推流程、群里公布），其余 N 人各演 1 角色。**必须满 N 人**才能开始（N=角色数）。
- **单幕**：开场动画(剧本名) → 每人读自己的角色卡 → 线索随幕自动公开(`autoClues`，不搜证) → 主持人「一键复制」把开场白+群活动+线索发微信群 → 群里讨论/互指 → 回小程序投票 → **投满才能公布真相**。
- **结算分流**：玩家端只看「去群里等待主持人公布真相」；主持人端「一键复制(结果+真相+信)」发群公布。

---

## 一、设计铁律（最重要，巷往/拾光迭代出来的）

1. **线索全是物证、一人一条**：N 个嫌疑人就 N 条线索，每条是一件**现场物证**，各指一个人。
2. **动机不进线索**：动机三人都有，放在「背景 / 关系网 / 角色卡」里，**绝不写进线索**。
3. **物证不点名、不给答案**：线索只描述客观物证，不能出现"谁手上有伤""跟 X 摊牌的录音"这类广播式 giveaway（否则秒破）。凶手的破绽靠**身份去推**（如"货仓的东西只有进货的人会拎上台"）。
4. **三人物证等重**：每条物证都要把对应的人**放进现场/坐实嫌疑**，看着都像；谁也别明显比谁弱（弱了就加强，如给 阿May 补"尸体边沾血、保险柜按出血指印"）。
5. **凶手在真相里才锁定**：靠"凶器来历 + 他的破绽（擦/藏/伤）"三样对上一个人；其余两人的物证只证明"到过、闹过"，不是杀人证据。凶手的致命细节（如割破的手）**只放凶手角色卡 + 真相收尾**，不进公开线索。
6. **时间线闭环**：被害人「一个一个单独算账」（造密室+无目击）；明确**谁先发现尸体**；每人对外一句「我走/到时他还活着 or 已死」——**凶手撒谎、其余属实**，制造对峙。

---

## 二、角色卡套路（`actStories[0]`，玩家看到的）

5 段固定结构，段落间 `\n\n`（空行间距），重点句用 `**…**` 高亮：

1. **我是谁**（身份+背景一句）
2. **和被害人的恩怨**（动机）
3. **今晚行踪 + 我的物证/嫌疑**（凶手在这段写**作案手法+毁证**；平民写"我的物证能解释成没杀人"）
4. **甩锅另两人**（各一句，点他俩各自的嫌疑）
5. **结尾伏笔**（括号，被害人最近反常……给反转/结局铺垫；三人保持一致）

- **高亮约定**：每人 3 句 = 自己最关键的 1 句 + 甩锅的 2 个方向（`**…**` 必须成对）。
- **凶手卡额外**：给一句「对外撒谎口径」（如"算完账我就出来了，他还好好的"）。
- 名字在显示时会被替换成玩家昵称（`makeNamer`），写原角色名即可。

---

## 三、真相套路（小说体，整体视角）

**纯叙事讲清整个故事，不做"线索分析①②③"**（试过分析体，最后定为小说体）：

> 被害人其人（性格/为何招恨） → 三人各自的把柄+动机 → 那一夜一个一个算账（按顺序写每人做了什么：争吵/作案/发现尸体） → 自然揭出凶手 → **反转 / 主题句收尾**。

- 揭凶要自然（"……动手的，是 XX"），不用罗列证据。
- **反转**可走两条：①被害人**遗书** `letter`（巷往：癌症+原谅+留店）；②让真相**自己承担**反转、`letter` 留空（拾光）。二选一。
- **主题句收尾**点睛（拾光："有些位置，要等亲手失去，才看清那本就是你最好的地方"）。

## 四、数据字段清单（`utils/scripts.js`）

```js
const SCRIPT_XXX = {
  id, title, subtitle, tag,
  cover: { emoji, gradient, image: "" },      // 没图就留空，显示 emoji
  minPlayers, maxPlayers,                       // = 角色数
  duration, autoClues: true,
  intro,                                        // 首页选本简介 3~4 句
  worldview,                                    // 开场世界观长文（含"一个一个算账""谁先发现"）
  victim: { name, desc },                       // ★ 被害人名(结算署名用，已动态)
  relations: [ "A → B：恩怨/动机", ... ],       // 公开关系网=动机池
  characters: [ {                               // N 个
    id, name, gender:"", title(身份), isMurderer,
    persona:"", timeline:[], secret:"",
    actAsks: [["盘问别人一句"]],                // 单幕用 [0]
    actTasks: [["本幕要做的事", ...]],
    objective:"",
    actStories: [ "见§二的 5 段结构" ],
  }, ... ],
  acts: [ {                                     // 单幕只 1 个
    title, narration,                           // 念白(含一个一个算账+谁先发现)
    clueIds: ["x1","x2","x3"],
    hostPrompts: ["冷场直接问的一两句"],
    host: { cast:[一句话介绍每人+嫌疑], story:开场白, activities:[群活动4步] },
  } ],
  clues: [ { id, name, icon, text(纯物证) }, ... ],  // 一人一条
  truth: { murderer(角色id), title, text(小说体), letter("" 或 遗书) },
};
```

群活动固定 4 步：①轮流自我介绍+解释线索 ②群里初投 ③当众反驳 ④回小程序最终投票。

## 五、接入步骤（两文件必须同步！）

1. `utils/scripts.js`：写 `SCRIPT_XXX` → 加进 `SCRIPTS` 数组 → 加进 `module.exports.list`（首页展示）。
2. `cloudfunctions/game/index.js` 的 `SCRIPT_META`：
   ```js
   xxx: { charIds:[角色id...], murderer:角色id, actCount:1, cluesByAct:[[线索id...]], searchPerAct:1 },
   ```
   charIds / murderer / cluesByAct 必须和 scripts.js 一致。
3. ⚠️ **云函数必须重新部署**（右键 `cloudfunctions/game` →「上传并部署」），否则建房会回退成默认 6 人本（症状：要 6 人、没有信）。
4. 自检：建房后房间页显示「需 N 名玩家」= N 对了就成功。

## 六、你给我什么，我就能生成

最少给我：
1. **场景 & 被害人**：在哪、被害人是谁、什么性格、怎么死的（手法/凶器、密室怎么成立）。
2. **N 个角色**：每人——身份、和被害人的恩怨(动机)、今晚行踪、是不是凶手。
3. **凶手手法**：凶器/手段、怎么毁证、什么破绽能最终锁定他（且这破绽**别写进公开线索**）。
4. **反转/主题**：被害人遗书揭什么，或真相结尾要点的那句题。

或者——**直接丢一个现成剧本/小说给我**，我按本文套路全拆解改写：N 条物证线索（一人一条、动机剥离）、角色卡（5段+段距+高亮+撒谎口径）、当晚时间线、小说体真相、两文件同步。
