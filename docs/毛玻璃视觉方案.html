毛玻璃 信息不对称的形状 · 抽象 · 零场景

剧本杀的本质是信息不对称——每个人只知道自己的那部分。毛玻璃就是它的物理形态：看得见人影，看不清脸；隔着雾能感到"别人也有故事"，但只有你的那块被擦开。三个动作贯穿全程：起雾（隐藏）、擦拭（获得）、碎裂（真相）。没有场景、没有道具，任何本子都不出戏。

公布真相 · 碎雾

为什么毛玻璃和剧本杀是同一个东西

  • 隐藏 = 起雾：候场时所有人是雾里的人影；剧情页你能"看见"别人也有故事（雾墙后的模糊字迹），但读不了——这正是剧本杀的心理张力：我知道你藏着事。
  • 获得 = 擦拭：就位是擦亮自己；你的剧情是"已为你擦开·仅你可见"的一块；指认是擦开你怀疑的人。所有"知情"动作统一成一个手势。
  • 真相 = 碎裂：公布真相时玻璃"啪"地裂开，所有人影同时清晰——信息不对称结束的瞬间，被做成了全场最大的动效。
  • 题材通吃：雾没有情绪立场——悬疑本的雾是紧张，欢乐本的雾是"猜猜我是谁"；tag 含恐怖可以在雾上加一个缓慢划过的手印。

与真实流程逐页对应

  • room：ready 字段 → 擦痕划过+盖板消失；「等待玩家准备」→「雾里还有 N 个人影」。
  • game：开局播一次「哈气起雾→擦开见剧名」（transShow 换皮）；玩家屏 = 雾墙+擦开的故事卡（别人的故事骨架线是纯装饰 div，不涉及真数据）；主持人发群卡照旧。
  • voting：点谁擦开谁（可改票 = 上一个重新蒙雾）；已投进度照旧。
  • result：主持人「公布真相」→ 裂纹画开+白闪+全部清晰+结果行升起；玩家等待页 = 「雾还没散，等主持人揭晓…」。

技术与降级

  • mockup 用「模糊头像 + 半透明盖板」模拟磨砂，不依赖 backdrop-filter——小程序端直接照搬就能跑，兼容性无忧。
  • 想要"真·透过玻璃"的效果（盖板后内容实时模糊），Skyline 渲染下可用 backdrop-filter，WebView 下自动退回本方案的模拟做法——两档效果都成立。
  • 擦痕/水珠/裂纹/白闪全是 CSS keyframes + 一段 SVG，性能与光影版同级。
